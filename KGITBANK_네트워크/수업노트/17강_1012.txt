☆Access List
-ACL : 정책을 기반으로 데이터의 접근 제어 기능.
	*사용목적 : 보안적 기능, 데이터 분류
	*적용위치 : 네트워크 출입구(Gateway)
	*종류 : 숫자형(Number), 문자형(Name)

ACL1
ACL2

ACL AAA
ACL TEST

-Router에서는 ACL(Access Control List) 을 사용하여 트래픽 식별, 필터링, 암호화, 분류, 변환 작업을 수행할 수 있다.

-Router를 경유하는 Packet을 Filtering 한다. 
-허용(permit), 거부(deny)

#종류
*Standard Access List :  출발지 주소만을 가지고 필터링을 함
Ex) A학생이면 무조건 다 허용됨 (게임,구글 다됨)
	-Source Address를 검사한다
	-검사결과에 따라 전체 Protocol Suite에 대한 Packet출력을 허용하거나거부한다.

*Extended Access List : 출발지, 목적지,프로토콜, 포트 번호 사용해 필터링. (And 연산이어야 됨)
Ex) A학생. 네이버는 수업을 위해 허용, 게임은 허용x 
	-Source Addres와 Destination Address를 모두 검사한다.
	-특정 Protocol, port 번호, 다른 매개변수를 검사하여 유연하게 제어 가능하다. 


*들어오면 INBOUND(들어올때 적용받는 규칙), 나가면 OUTBOUND(나갈떄 적용받는 규칙)
 -외부에서 내부로 : INBOUND
 -내부에서 외부로 : OUTBOUND
window + r -> control 입력 -> 시스템 보안 -> 고급 설정 -> 인바운드규칙 & 아웃바운드 규칙

-ACL은 좁은 범위부터 넓은 범위로 지정해야 된다.(피라미드 형식)

-Wildcardmask  : 해당 네트워크 주소의 매칭 여부를 가름. (0과 대응 되는 비트 검사해, 1은 무시해)
-Subnetmask : 네트워크 주소에서 N-ID, H-ID를 식별하는 주소

ACL 설정 순서(F/W 구성)
1. 정책 분류 : 정책 매칭 대상 분류
2. 정책 설정 : 정책 대상을 ACL등으로 설정
3. 정책 적용 : 정책 설정 입출입 인터페이스에 적용 

Step1(정책 설정)
(config)#access-list access-list-number {permit | deny} source [mask]
-Access-list-number : 1~99(Standard), 1300~1999사이 번호


Step2(정책 적용)
(config-if) #ip access-group access-list-number {in|out}
